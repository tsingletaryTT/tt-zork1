# Demo Suite Redesign — 2026-04-29

## Context

The `demos/` directory currently has four scripts (`demo-stage1.sh` through `demo-stage4.sh`) plus `record-all.sh` and `README.md`. The Stage 4 script shows a human switching from classic to remix mode live. This redesign replaces it with two purpose-built demos and updates all scripts to reference the single best available model (Llama-3.3-70B-Instruct).

**Audience:** GitHub README viewers first (self-narrating, works without a presenter), internal team and conference audiences second.

## demo-hybrid.sh — Human Plays, Remix On

**Goal:** Short, punchy (~90 seconds). Shows LLM remix transforming Z-machine output into richer prose, without the AI playing for you. Suitable as a quick README embed.

**Constraints:**
- Scripted (piped commands), runs unattended for recording
- Sets `ZORK_LLM_MODEL=meta-llama/Llama-3.3-70B-Instruct`
- Uses `python play.py --stage sim --game game/zork1.z3 --remix`
- Commands: `look`, `open mailbox`, `take leaflet`, `read leaflet`, `go north`
- Sufficient sleep between commands for remix LLM to respond

**What viewer sees:** Classic Zork prompts, LLM-rewritten room descriptions, the same location in a richer voice.

## demo-ai.sh — The Showpiece

**Goal:** The full experience. Runs unattended. AI plays the game with experimental persona + full TUI + remix layer. Pure chaos from turn one.

**Command:**
```bash
ZORK_LLM_MODEL=meta-llama/Llama-3.3-70B-Instruct \
  python play.py \
    --stage sim \
    --game game/zork1.z3 \
    --remix \
    --tui \
    --persona experimental \
    --turns 30
```

**Persona choice:** Experimental all the way — tries unusual commands, pushes edges, remix voice makes absurd responses vivid. Natural arc: disoriented discovery → increasingly weird choices → poetic or absurdist output from the remix layer.

**TUI panes during recording:**
- Left pane: game output + AI-generated commands scrolling
- Right pane: cycles THINKING (streaming tokens) → ART (ASCII room art generated by 70B)
- Status bar: shows `stage=sim`, `remix=on`, `turns` count incrementing

**Prerequisites:** tt-inference-server running with Llama-3.3-70B-Instruct at `http://localhost:8000/v1`.

## Model Wiring

`ZORK_LLM_MODEL=meta-llama/Llama-3.3-70B-Instruct` is set at the top of each script that uses the remix layer. The router's autodetect (`remix/router.py`) already collapses all roles (input mapping, output remix, art, postcards) to the single detected model when one is running, so no code changes are needed in the router.
